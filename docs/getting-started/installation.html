Installation

Get Huntarr running on Windows, Linux, or macOS in minutes.

Jump to: Windows  ·  Linux  ·  macOS  ·  Environment Variables  ·  Updating

Requirements

  • Python 3.10+ — required for running the application
  • pip — Python package manager
  • 512 MB RAM minimum (1 GB+ recommended)
  • Network access to your *arr instances (Sonarr, Radarr, etc.)
  • A modern web browser for the Huntarr UI

Windows Installation

Run from Source

  1. Install Python 3.10+ (check "Add to PATH" during install).
  2. Install Git for Windows.
  3. Open PowerShell and run:
git clone https://github.com/emiliomm/huntarr-custom.git
cd Huntarr.io
pip install -r requirements.txt
python main.py

Access Huntarr at http://localhost:9705.

macOS Installation

Run from Source

brew install python@3.12 git
git clone https://github.com/emiliomm/huntarr-custom.git
cd Huntarr.io
pip3 install -r requirements.txt
python3 main.py

Access Huntarr at http://localhost:9705.

Linux Installation

Run from Source with systemd

sudo apt update && sudo apt install -y python3 python3-pip git
git clone https://github.com/emiliomm/huntarr-custom.git /opt/huntarr
cd /opt/huntarr
pip3 install -r requirements.txt

Create a systemd service file at /etc/systemd/system/huntarr.service:

[Unit]
Description=Huntarr Media Hunter
After=network.target

[Service]
Type=simple
User=huntarr
WorkingDirectory=/opt/huntarr
ExecStart=/usr/bin/python3 /opt/huntarr/main.py
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable --now huntarr
sudo systemctl status huntarr

Environment Variables

Variable Default Description
TZ UTC Timezone for scheduling and log timestamps (e.g., America/New_York).
HUNTARR_CONFIG_DIR Platform Dependent Directory where database and settings are stored.
HUNTARR_PORT 9705 Internal web UI port.
HUNTARR_LOG_LEVEL INFO Log verbosity. Options: DEBUG, INFO, WARNING, ERROR.

Updating Huntarr

From Source

cd /opt/huntarr
git pull
pip3 install -r requirements.txt
# Restart the process (or systemd service)
sudo systemctl restart huntarr

Post-Installation

After installation, open http://your-server:9705 in your browser. On your first visit you'll be greeted by the Setup Wizard, which guides you through creating your admin account, connecting your first *arr app, and configuring basic hunt settings.

Troubleshooting

Port conflict

If port 9705 is already in use, set HUNTARR_PORT environment variable to a different value.

Permission errors

Ensure the user running Huntarr has write permissions to the configuration directory.

Can't reach *arr apps

Verify network connectivity between Huntarr and your *arr instances.
